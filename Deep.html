Análisis de Clasificación de Personajes de Los Simpson

Proyecto de Deep Learning

Este proyecto analiza diferentes arquitecturas de redes neuronales convolucionales para la clasificación automática de personajes de la serie Los Simpson.

Se han implementado y evaluado cinco arquitecturas diferentes para determinar cuál obtiene el mejor rendimiento en esta tarea específica de clasificación de imágenes.

Interpretación

Las arquitecturas evaluadas presentan diferentes enfoques para la clasificación de imágenes:

  • CNN Básica: Arquitectura simple pero efectiva, con un buen balance entre complejidad y rendimiento.
  • VGG-Like: Modelo más profundo con muchos más parámetros, pero que no se traduce en mejor rendimiento para este conjunto de datos específico.
  • MobileNet-Like: Arquitectura que utiliza convoluciones separables en profundidad para reducir el número de parámetros manteniendo un rendimiento aceptable.
  • Transfer Learning: Los modelos pre-entrenados (MobileNetV2 y EfficientNet) no lograron superar a los modelos entrenados desde cero, posiblemente debido a la diferencia entre las imágenes naturales de ImageNet y los dibujos animados de Los Simpson.
Arquitecturas CNN para clasificación de imágenes

Las redes neuronales convolucionales (CNN) son particularmente efectivas para la clasificación de imágenes debido a su capacidad para detectar patrones espaciales. Las arquitecturas evaluadas en este proyecto varían en su complejidad y enfoque.

Mientras que las redes más profundas como VGG suelen funcionar bien en conjuntos de datos grandes y complejos, para tareas específicas como la clasificación de personajes animados, una arquitectura más simple y ajustada puede ofrecer mejores resultados con menos recursos computacionales.

Entrenamiento

Parámetros de Entrenamiento

Épocas 8
Batch Size 64
Optimizador Adam con learning rate scheduler
Learning Rate Inicial 0.001
Función de Pérdida Categorical Crossentropy
Se utilizó un scheduler de tasa de aprendizaje para reducir gradualmente el learning rate a medida que avanza el entrenamiento, lo que ayuda a alcanzar un mínimo más estable.

Callbacks Utilizados

  • EarlyStopping: Detiene el entrenamiento si la pérdida de validación no mejora durante 5 épocas.
  • ReduceLROnPlateau: Reduce la tasa de aprendizaje en un factor de 0.5 si la pérdida de validación no mejora durante 3 épocas.
  • ModelCheckpoint: Guarda los mejores pesos del modelo según la precisión de validación.

Fine-tuning del Mejor Modelo

Proceso de Fine-tuning

Se realizó un fine-tuning de la CNN Básica para mejorar su rendimiento:

  • Se redujo la tasa de aprendizaje a 0.0001
  • Se aplicó un scheduler de tasa de aprendizaje exponencial
  • Se entrenó durante 5 épocas adicionales
  • Se utilizaron los mismos callbacks que en el entrenamiento original

Impacto de Técnicas Aplicadas

Interpretación

Las técnicas que más impacto tuvieron en la mejora del rendimiento fueron el aumento de datos (data augmentation) y la normalización por lotes (batch normalization). Estas técnicas ayudaron a mejorar la generalización del modelo y estabilizar el entrenamiento.

Conclusiones

Hallazgos Principales

  1. Arquitectura más efectiva: La CNN Básica demostró ser la más efectiva para esta tarea específica, con una precisión de test del 63.13%, superando a modelos más complejos.
  2. Modelos de transfer learning: Sorprendentemente, los modelos pre-entrenados (MobileNetV2 y EfficientNet) no superaron a los modelos entrenados desde cero, posiblemente debido a la diferencia entre las imágenes naturales de ImageNet y los dibujos animados de Los Simpson.
  3. Complejidad vs. rendimiento: No siempre más parámetros implican mejor rendimiento. El modelo VGG-Like, con más de 45 millones de parámetros, tuvo un rendimiento inferior a la CNN Básica con solo 2.5 millones de parámetros.
  4. Fine-tuning: El fine-tuning mejoró el rendimiento del mejor modelo en un 1.72%, llevando la precisión final a 64.85%.
  5. Personajes principales: Los personajes con más presencia en la serie (Homer, Bart, Marge) obtuvieron mejores resultados de clasificación, posiblemente debido a la mayor cantidad de imágenes en el conjunto de datos.

Recomendaciones

Para mejorar el rendimiento
  • Aumentar la resolución de las imágenes para capturar más detalles
  • Incrementar el conjunto de datos con más imágenes de personajes minoritarios
  • Aplicar técnicas de ensemble (combinación de modelos)
  • Realizar una búsqueda exhaustiva de hiperparámetros
  • Probar arquitecturas híbridas que combinen elementos de diferentes modelos
Para implementación en producción
  • Convertir a TensorFlow Lite para dispositivos móviles
  • Implementar una API REST con TensorFlow Serving para aplicaciones web
  • Aplicar cuantización para reducir el tamaño del modelo
  • Establecer un sistema de monitoreo para evaluar el rendimiento continuo
  • Implementar un sistema de retroalimentación para seguir mejorando el modelo

Lecciones Aprendidas

Principales Aprendizajes
  • Una arquitectura más simple puede superar a modelos más complejos cuando está bien adaptada al problema específico.
  • El preprocesamiento y aumento de datos son tan importantes como la arquitectura del modelo.
  • Los modelos pre-entrenados no siempre son la mejor opción, especialmente cuando el dominio de las imágenes es muy diferente al conjunto de entrenamiento original.
  • La normalización por lotes (batch normalization) mejora significativamente la estabilidad del entrenamiento.
  • El balance entre complejidad del modelo y rendimiento es crucial para obtener resultados óptimos.
  • El fine-tuning puede mejorar el rendimiento incluso en modelos que no son de transfer learning.

Conclusión Final

Este proyecto demuestra que para la clasificación de personajes de Los Simpson, una CNN relativamente simple puede obtener buenos resultados, superando incluso a arquitecturas más complejas y modelos pre-entrenados. La elección de la arquitectura adecuada, junto con un buen preprocesamiento de datos y técnicas de regularización, son factores clave para el éxito en tareas de clasificación de imágenes específicas.
